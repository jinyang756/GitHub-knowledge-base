# 国泰海通证券知识库

## 国泰海通证券知识库

国泰海通证券知识库是由国泰海通证券研究所倾力打造的专业金融知识平台，采用docsify框架构建，支持全文搜索、响应式布局、高级视觉效果等功能，为投资者、研究人员和从业人员提供全面、权威、及时的金融市场信息。

本知识库详细介绍了国泰海通证券的研究成果、市场分析、投资策略和专业知识，旨在帮助用户更好地了解金融市场动态，做出明智的投资决策。

## 功能特点

1. **高级视觉设计**：采用"深空蓝+金色点缀"专业金融色彩体系，融合动态背景、粒子效果和光效系统，打造沉浸式视觉体验
2. **智能搜索功能**：支持全文搜索，提供实时搜索建议和高亮显示，快速定位所需信息
3. **响应式布局设计**：完美适配PC端、平板和手机等多种设备，随时随地获取专业知识
4. **路径别名功能**：为常用文档配置了简化访问路径，方便用户快速访问
5. **数学公式支持**：集成MathJax，支持LaTeX数学公式渲染，满足技术文档和金融分析需求
6. **代码复制功能**：一键复制代码块内容，提升用户体验
7. **缓存控制**：配置合理的缓存策略，确保获取最新内容同时优化加载速度

## 快速开始

### 本地预览

1. 确保安装了Node.js环境（推荐v14.0及以上版本）
2. 安装docsify-cli工具：`npm i docsify-cli -g`
3. 在项目根目录下运行：`docsify serve docs`
4. 打开浏览器访问：`http://localhost:3000`

### 使用批量脚本工具

项目提供了便捷的批量处理脚本，可直接双击运行：
- `转换DOCX到MD.bat`：批量将DOCX文档转换为Markdown格式
- `更新指定目录Alias配置.bat`：批量更新指定目录的docsify别名配置，方便短链接访问

## 内容更新指南

### 1. 准备新内容

新内容应先以Word文档(DOC/DOCX)格式编写，确保内容完整、格式规范。内容需包含：
- 清晰的标题和副标题
- 结构化的内容组织（使用标题层级）
- 必要的图表和数据
- 引用来源标注

### 2. 格式转换

使用项目提供的转换工具将Word文档转换为Markdown格式：

#### 单个文件转换
- 打开命令提示符
- 导航到项目根目录
- 运行：`python docx_to_markdown.py 输入文件.docx 输出文件.md`

#### 批量转换
- 双击运行`转换DOCX到MD.bat`脚本
- 按照提示输入源文件夹和目标文件夹路径
- 系统将自动批量转换所有DOCX文件

### 3. 内容优化

转换后的Markdown文件可能需要进一步优化：
- 检查标题层级是否正确
- 调整表格格式确保显示正常
- 优化图片引用路径
- 添加必要的标签和样式

### 4. 添加到知识库

将优化后的Markdown文件复制到docs目录下对应的子目录中，例如：
- 政策解读内容 -> `docs/policy/`
- 行业分析内容 -> `docs/industry/`
- 投资策略内容 -> `docs/strategy/`

### 5. 更新导航配置

更新侧边栏导航配置，确保新内容可以被正确访问：

1. 打开`docs/_sidebar.md`文件
2. 在相应的分类下添加新文件的链接，格式为：`* [标题](/目录/文件名.md)`
3. 保存更改

### 6. 更新别名配置（可选）

对于重要或常用的文档，可以配置路径别名以简化访问：

1. 双击运行`更新指定目录Alias配置.bat`脚本
2. 按照提示输入要更新的目录路径
3. 系统将自动更新`docs/index.html`中的alias配置

### 7. 本地预览

在提交更改前，建议先进行本地预览以确保内容显示正常：

1. 安装Node.js环境（推荐v14.0及以上版本）
2. 安装docsify-cli工具：`npm i docsify-cli -g`
3. 在项目根目录下运行：`docsify serve docs`
4. 打开浏览器访问：`http://localhost:3000`

### 8. 提交更改

确认内容无误后，提交更改到版本控制系统。

## 内容规范

为保证知识库内容的一致性、专业性和权威性，请严格遵循以下内容规范：

### 标题规范
- 使用清晰的标题层级（#、##、###等）组织内容，层级不宜过深（建议不超过4级）
- 一级标题（#）仅用于文档主标题
- 二级标题（##）用于主要章节
- 三级标题（###）用于章节内的子主题

### 内容组织
- 每个文档应有明确的主题和结构
- 开篇应有简短的内容介绍
- 正文内容应层次分明，重点突出
- 结尾可包含总结或参考文献

### 样式规范
- 对于核心数据和重要结论，使用`<span class="data-number">关键数据</span>`样式进行高亮标记
- 对于内容分类，使用`<span class="tag">#标签1 #标签2</span>`格式添加标签
- 保持图片、表格、代码块等元素的格式一致性

### 图片处理
- 图片应放置在`docs/images/`目录下对应分类的子目录中
- 图片引用路径使用相对路径，如`![图片描述](../images/分类/图片名.jpg)`
- 确保图片清晰可读，大小适中

### 合规要求
- 所有内容必须符合国家法律法规和金融行业相关规定
- 引用的数据必须注明来源和时间，确保信息的准确性和可追溯性
- 避免使用敏感词汇和不实信息

## 工具脚本说明

知识库提供了一系列辅助工具，方便内容管理和格式转换：

### 文档转换工具

- **docx_to_markdown.py**: 将DOCX格式文档转换为Markdown格式，保留基本格式和图片引用
  - **功能特点**：自动转换标题层级、列表、表格和引用格式
  - **使用方法**：`python docx_to_markdown.py 输入文件.docx [输出文件.md]`
  - **依赖要求**：需要安装python-docx库 (`pip install python-docx`)
  - **示例用法**：
    ```bash
    # 默认输出名称（与输入文件同名的.md文件）
    python docx_to_markdown.py 报告.docx
    
    # 指定输出名称
    python docx_to_markdown.py 报告.docx 我的报告.md
    
    # 直接转换到知识库目录
    python docx_to_markdown.py 新文档.docx docs/分类/新文档.md
    ```
  - **注意事项**：
    - 主要支持标准Word文档格式，特别复杂的文档可能需要手动调整
    - 对于特别长的文档，转换可能需要一定时间

- **convert_doc_to_docx.py**: 将旧版DOC格式转换为DOCX格式，便于后续处理
  - **功能特点**：批量转换旧版Word文档

### 配置管理工具

  - **功能特点**：自动扫描指定目录并更新alias配置

- **标题层级**：使用清晰的标题层级（#、##、###等）组织内容，层级不宜过深
- **重点标记**：对于核心观点、重要数据等，使用<span class="data-number">关键数据</span>样式进行高亮标记
- **结构化表达**：尽量使用列表、表格等结构化形式呈现内容，提升可读性
- **数据来源**：引用的数据必须注明来源和时间，确保信息的准确性和可追溯性
- **代码规范**：技术指标公式和代码示例应遵循行业标准规范，添加必要的注释说明
- **专业术语**：使用规范的金融专业术语，避免使用歧义或不专业的表述
- **格式统一**：保持文档格式的一致性，包括字体、字号、行距、缩进等
- **内容审核**：发布前必须经过严格的内容审核，确保信息的真实性和合规性

## 贡献指南

国泰海通证券知识库欢迎专业研究人员和行业专家进行贡献，具体流程如下：

1. Fork本仓库到个人GitHub账号
2. 创建新的分支：`git checkout -b feature/your-feature-name`
3. 在docs目录下的对应分类中添加或修改相关文档内容
4. 确保内容符合文档编写规范（见下文）
5. 提交更改：`git commit -am 'Add some feature'`
6. 推送到远程：`git push origin feature/your-feature-name`
7. 创建Pull Request并等待审核

**内容审核流程：**
提交的内容将经过严格的合规性检查和专业性评审，确保知识库内容的权威性和准确性。

## 部署说明

知识库支持以下部署方式：

### GitHub Pages部署

1. 在项目的Settings中，找到GitHub Pages选项
2. 选择docs目录作为发布源
3. 选择合适的分支（通常为main或master）
4. 保存配置后，稍等片刻即可通过GitHub Pages访问

### 本地服务器部署

1. 确保安装了Node.js环境
2. 在项目根目录运行：`npm i docsify-cli -g && docsify serve docs`
3. 可通过参数指定端口：`docsify serve docs --port 8080`
